Transforming notes to blog posts

June 12, 2020
Table of Contents
  • Why?
  • Solution
  • Handling citations

Why?

I keep a lots of notes containing different ideas and thoughts, but they are not well maintained. As a result, they are not very useful for myself or for others. It would be great if if these notes can be polished and made available online.

Currently my personal site is made with Jekyll. However, it’s quite difficult to automatically generate org notes to blog posts in Jekyll. Need to try something else.

Solution

I use ox-hugo to generate new blog posts in markdown. The concept is quite simple. The org files are first exported to markdown files. Then hugo builds the html site from these markdown files, which is then deployed using Github pages.

In particular, for each org note that is to be published, fill in the front matter as shown below. When satisfied with the note, use `org-export` (C-c C-e H h) to generate the markdown post. After doing this, there should be a new mardown file in `publish\content\posts` with the same name as the org file but in hugo markdown format.

#+TITLE: Transforming note to blog posts
#+HUGO_SECTION: posts
#+HUGO_CATEGORIES: blog hugo
#+HUGO_BASE_DIR: publish
#+DATE: 2020-06-12
#+toc: headlines 2

To preview the site

cd ~/org/publish && hugo serve

To build and deploy the site

cd ~/org/publish && ./deploy.sh
